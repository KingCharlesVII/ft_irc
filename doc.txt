TCP:

Transmission control protocol: this protocols makes sure the data arrives sequentiqlly and error-free.
It cuts a byte stream into segments wich size depents of the MTU (maximum transmission unit). FOr most othe the ethernet network, MUT is 1500 bytes.
So the maximum segment TCP MSS(Maximum segment size) is about 1460 bytes. The size of data segment plus the one of the IP header shoul be less thant MTU.
MMS should be announced by each the communicating computers.

ACK mean aknoledgement. TCP confirms the reception of the bytes, not the packets.

If a packet is lost: 1 2 3 4, reveivec: 1 3 4. It answers ACK = 2.
Every segment owns a ckecsum, the destination calculates this value. It it differs, the packet is thrown then TCP will wait a retranmission.
If no packet arrives, retransmission and a retransmission is raised. If the tiemout is too long, TCP uses Fast retransmit: 1 2 3 4 5.
It receives 1 3 4 5 so it answers ACK=2 ACK=2 ACK=2.

Socket:

We get this file descriptor with Socket.
We can also do this with read and write, but send() and recv() offer a greater control.
We can use stream socket to receive packages in the same order.
We can unser Datagram sockets so the packages arrive in any order. They are called connectionless. The data withing the packet will be error-free if it arrives.
Datagram socket also use IP  for routing but they don't unser TCP, they use the User Datagram Protocol.
They are called connectionless because we don't need to maintain an open connection as we do with stream sockets. We builda packet, slap an IP header on it with destinaion information and send it out.
NO connection required. They are used when a TCP stack is unavailable. They are used with fttp (trivial file transfer protocol, little brother of FTP), dhcp, multiplayers ga;es and so on.
We can use this scoket for games since we ignore the droped packets and perhaps truc to compensate for them (lag). THus, w use this protocol for seepd. If we send messages in chats, it does not matter until we
update 40 positions at once.
All the other rely on this two models (HTTP, HTTPS, SSH, DNS, game, streaming)
